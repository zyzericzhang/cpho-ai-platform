# 模块规格：文章编辑器

## 模块定位

文章编辑器用于创建和编辑文章 / 文档。

第一版尽量使用富文本编辑器，而不是 Markdown + 预览。原因是初级用户不一定会 Markdown，边修改边预览也可能比较困难。

第一版不需要做成完整 Google Docs，但应该尽量是直接编辑、直接看到效果的形式。

## 第一版能力

文章编辑器需要支持：

- 创建文章 / 文档
- 编辑文章内容
- 保存文章
- 设置公开 / 私有
- 选择知识点 / 模型标签
- 关联题目
- 在正文任意位置嵌入题目内容并跳转
- 从 AI 解题会话中插入内容

预览是否需要独立存在，取决于富文本编辑器实现。

## 题目嵌入

文章正文中任意位置可以嵌入题目内容。

嵌入的题目可以跳转到题目详情。

题目具体显示成完整题面还是题目卡片，目前未最终决定。

当前只确认：

- 文章正文中可以插入题目引用
- 读者可以通过这个引用跳转到题目详情

## AI 对话内容插入

用户可以把 AI 解题会话中的内容插入文章。

这里不是把 AI 解题会话本身放进个人学习库，而是把会话中的部分内容作为文章内容插入。

## 数据需求

文章编辑器需要维护：

- 文章内容
- 富文本内容
- 公开状态
- 知识点标签
- 模型标签
- 嵌入题目 ID
- 关联题目列表
- 从 AI 会话插入的文本内容
- 保存时间
- 修改时间
- 作者
