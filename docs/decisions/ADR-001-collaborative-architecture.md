# ADR-001: 协作开发架构从“黑盒”转向“透明黑板”模式

## 状态
已接受 (Accepted)

## 日期
2026-05-10

## 背景
在之前的开发过程中，Agent 的执行过程往往是“黑盒”的，用户只能看到代码变化而无法理解背后的逻辑。同时，Agent 容易出现“长时健忘”，导致重复沟通和架构偏差。此外，依赖外部 Figma 链接增加了上下文复杂度和语言障碍。

## 决定
1. **引入三层记忆模型**：使用 `SOUL.md` (价值观)、`PROJECT_STATUS.md` (资产记忆) 和 `WORKING_CONTEXT.md` (短期上下文) 建立多维记忆。
2. **启用动态黑板**：以 `IMPLEMENTATION_LOG.md` 作为实时沟通枢纽，强制执行“写前必思 (Think-Before-Write)”协议。
3. **移除 Figma 依赖**：将 UI 事实来源转移到本地 `design/` 截图和 `docs/` 规范中。
4. **全中文协议**：所有高频交互文件和日志必须使用中文，以提高沟通精度。

## 后果
- **正面**: 提高了 Agent 的决策透明度，允许用户实时干预，减少了重复沟通。
- **负面**: 增加了一定的 Token 消耗用于维护状态文件，回合响应时间可能略微增加。
