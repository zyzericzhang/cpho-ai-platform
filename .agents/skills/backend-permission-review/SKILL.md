# Backend Permission Review

Use this skill for any backend-related implementation or review.

## Required Reading

1. `docs/permission-matrix.md`
2. `docs/backend-risk-checklist.md`
3. touched route/table/storage/provider files

## Checks

1. Can a student call admin-only APIs?
2. Can one user read another user's AI sessions?
3. Can one user read another user's library items?
4. Can one user read another user's uploaded files?
5. Can one user modify another user's article?
6. Are private queries scoped by `user_id` or `owner_id`?
7. Are Supabase RLS policies present and testable?
8. Are AI provider keys server-only?
9. Are secrets exposed through `NEXT_PUBLIC_`?
10. Are file type, size, and count validated server-side?
11. Are AI calls rate-limited or usage-limited?
12. Do logs avoid leaking secrets?
13. Are related problems/articles retrieved from database search rather than generated by the model?
14. Is "No standard answer, no AI solution" enforced server-side?

## Output

Return:

- critical blockers
- high risks
- medium risks
- missing tests
- required fixes
- manual verification steps
- merge recommendation
