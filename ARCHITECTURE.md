# ARCHITECTURE.md | 协作开发架构指南 (ECC 适配版)

本文件是为你（用户）编写的，旨在解释这套全新的“分层指令与实时协作”架构是如何运行的，以及你如何通过它高效地指挥 Agent。

---

## 1. 核心架构：三层记忆模型

为了解决 Agent 的“健忘”和“黑盒”问题，我们将系统指令和状态分成了三层：

| 文件 | 角色 | 存储内容 |
| :--- | :--- | :--- |
| **`SOUL.md`** | **长期灵魂** | 项目的红线、价值观、工作原则（基本不随任务改变）。 |
| **`PROJECT_STATUS.md`** | **资产记忆** | 项目目前已经实现了什么，有哪些技术债，现有的核心结论。 |
| **`WORKING_CONTEXT.md`** | **短期记忆** | 当前这几分钟、这几个回合我在干什么，刚才说到了哪里。 |

---

## 2. 实时协作逻辑：动态黑板 (The Blackboard)

**`IMPLEMENTATION_LOG.md`** 是我们的实时沟通中心：
- **🧠 思维链**：我在动手写代码前，会在这里详细解释我的思路。如果你觉得不对，可以在下面直接纠正我。
- **🛑 交互干预区**：这是你专用的“紧急制动阀”。你在这里写的任何中文指令，我都会在下一轮优先读取并调整方案。
- **🛠️ 执行轨迹**：像看直播一样追踪我正在操作哪个文件，遇到了什么报错。

---

## 3. 决策追踪：ADR (架构决策记录)

重要的技术决定（例如：为什么选择 OpenRouter 而不是直接调用 Gemini）将被记录在 `docs/decisions/` 下。这确保了当未来新的 Agent 进入项目时，不会因为不了解背景而推翻之前的正确决定。

---

## 4. 精准路由 (Instruction Routing)

为了减少 Token 浪费，我们采用了子目录指令模式：
- **`app/GEMINI.md`**：只包含与路由、前端交互相关的指令。
- **`lib/GEMINI.md`**：只包含与后端逻辑、算法、安全性相关的指令。
Agent 进入相应目录时会主动加载对应的规则，避免“鸡同讲讲”。

---

## 5. 如何高效地使用我？

1. **直接在日志中交互**：如果你想修改我的思路，直接去 `IMPLEMENTATION_LOG.md` 的干预区留言。
2. **检查我的成效预测**：我不会再回复“修改了第 10 行”，我会说“现在登录页面会有一个淡入效果”，请你根据这个成效来评估我的工作。
3. **无需提及 Figma**：我已经彻底忘记了 Figma，请直接给我截图、文档或描述你的视觉意图。

---

## 6. 开发者心法 (For Agents)

如果你是另一个 Agent 或 sub-agent，进入此仓库的第一件事是：
1. 读取 `SOUL.md`。
2. 读取 `PROJECT_STATUS.md` 确认资产。
3. 读取 `WORKING_CONTEXT.md` 确认当前进度。
4. 严格按照 `IMPLEMENTATION_LOG.md` 执行并更新。
